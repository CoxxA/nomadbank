# 安全策略

## 支持版本

| 版本 | 状态 |
| --- | --- |
| 最新的 v2.x 正式版 | 支持安全修复 |
| 较旧的 v2.x 正式版 | 建议升级到最新版 |
| v0.x、v1.x | 不支持 |
| `main` 和未正式发布的构建 | 开发版本，不承诺生产支持 |

维护者会通过新的 PATCH 版本发布安全修复，不会覆盖或替换已经发布的附件。版本发布与撤回原则见[维护与发布指南](docs/releasing.md)。

## 报告漏洞

请通过 GitHub Security Advisories 的 “Report a vulnerability” 私下报告。不要在公开 Issue 中披露可利用细节、真实 Token、数据库或账户信息。

报告请包含受影响版本、部署方式、复现步骤、可能影响和可行的缓解建议。维护者确认前请给出合理修复时间。

## 安全边界

- NomadBank 不连接银行，也不执行真实转账。
- 数据库包含密码哈希、会话和个人账户任务，应保护文件权限和备份。
- 应通过 HTTPS 暴露到公网，并确保代理设置 `X-Forwarded-Proto: https`。
- 不应在 NomadBank 中保存银行卡号、网银密码或短信验证码。
