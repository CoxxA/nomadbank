# 升级说明

## 从 v0.x 升级到 v2

v2 是破坏式重写，不支持旧数据库迁移。它使用新的 `nomadbank-v2.db`，不会读取或修改旧的 `nomadbank.db`。

升级步骤：

1. 停止旧版本并备份整个数据目录。
2. 固定使用 v2 镜像标签或发布二进制。
3. 启动 v2。
4. 打开初始化页面，创建唯一所有者。
5. 重新录入账户和策略。
6. 确认无误后再决定是否归档旧数据库。

旧数据不会被自动删除，因此可以随时用旧二进制和原备份恢复查看。

## v2 之后

公开发布后的 v2 小版本和次版本必须提供向前 schema 迁移。升级前应：

1. 阅读 Release Notes 和 CHANGELOG。
2. 停止写入并备份数据目录。
3. 拉取精确版本标签。
4. 启动并等待 `/health/ready` 成功。
5. 执行登录、查看账户和任务的冒烟检查。

除非对应 Release Notes 明确说明，否则数据库升级后不支持直接降级。恢复方式是旧程序配合升级前备份。
