# 仪表盘重设计方案（现代科技风 + 深色优先）

## 目标与范围
- 目标：在不改动后端 API 的前提下，重塑仪表盘视觉与信息架构，突出任务执行效率与操作便捷性。
- 风格：现代科技、深色优先、玻璃拟态、Bento Grid。
- 范围：仅前端仪表盘页面与全局主题/字体配置；保留现有数据接口与交互能力。

## 设计原则
- 首屏可读：关键 KPI 与图表在首屏完成信息传达。
- 数据先行：任务效率指标优先，其次是执行/日历与活动流。
- 轻量交互：不做位移动效，使用发光/描边/透明度过渡。
- 深色优先：默认暗色，但保留浅色模式与系统切换能力。

## 视觉系统
- **色板**
  - 背景：#0B1020 → #0F172A 渐变，轻噪点纹理。
  - 主要强调：#F59E0B（完成/达成）。
  - 数据强调：#22D3EE / #38BDF8（科技蓝）。
  - 危险：#F97316 或 #EF4444（避免紫色倾向）。
  - 文字：主文本 #E6EDF7，次级文本 #A7B0BF。
- **字体**
  - UI 文本：Fira Sans。
  - 数字/金额/时间：Fira Code（数据感）。
- **材质**
  - 卡片：`bg-slate-900/55` + 12–16px blur + `border-white/10`。
  - 阴影：柔和大半径，避免强对比硬阴影。

## 布局与信息架构
1) 顶部区：标题 + 时间范围切换（本周/本月/本年）。  
2) KPI 区（4 卡）：完成率、待处理、延后/逾期、账户数。  
3) 主视觉：本月每日任务量柱状图（主列）。  
4) 右侧辅列：迷你日历 + 最近活动时间线。  
5) 今日任务：主列下半区全宽卡片。  
6) 右下角浮动操作区：提醒/完成/推迟。

## 关键组件说明
- **KPI 卡**：统一标题/数字/说明结构，右上角 Lucide 线性图标。
- **柱状图**：Recharts `BarChart`，柱体圆角 4px，渐变填充；Tooltip 使用深色浮层。
- **今日任务**：最多展示 5 条，滚动可见；金额与状态徽章突出。
- **迷你日历**：压缩行高与字号，保留可点选任务日期。
- **活动时间线**：纵向时间线轨道 + 节点状态色；信息卡简洁对齐。
- **浮动操作区**：右下角固定胶囊按钮组，44px 触控尺寸，加载态替换图标。

## 数据与交互
- 时间范围切换驱动 KPI 与图表刷新（共享 state）。
- 图表数据优先由日历/任务聚合计算，避免新增 API。
- 空态统一用“—”与短文案说明，不用 0 误导。
- 失败时 toast 提示 + 局部重试按钮。

## 可访问性与响应式
- 文字对比度满足 4.5:1，焦点态明显。
- 触控目标 ≥ 44px，键盘可达。
- 断点：375/768/1024/1440 适配；右侧辅列在小屏下移至主列后。

## 实施清单
- 更新主题：`frontend/src/styles/theme.css`（暗色优先与新色板）。
- 更新字体：`frontend/index.html` + `frontend/src/config/fonts.ts` + `@theme inline` 字体变量。
- 更新默认主题：`frontend/src/context/theme-provider.tsx`（默认 dark）。
- 重构仪表盘布局与组件：`frontend/src/features/dashboard/index.tsx`。
- 新增/复用图表与时间线展示组件（局部组件即可，不必通用化）。
