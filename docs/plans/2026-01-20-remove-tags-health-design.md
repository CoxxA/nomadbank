# 移除标签与健康度功能设计

## 背景与目标
本次需求要求彻底移除账号管理中的“标签”和“健康度”功能，覆盖前端、后端、数据库与文档，避免任何遗留入口或数据结构。

## 范围
- 后端：删除标签相关 API、存储层逻辑与模型；银行列表响应不再包含 `tags` 与 `last_exec_date`。
- 前端：移除标签管理 UI/交互、标签筛选与展示；移除健康度计算与展示。
- 数据库：删除 `tags` 与 `bank_tags` 表；健康度无独立字段，仅移除相关输出。
- 文档：同步清理 README/CLAUDE/设计文档中的标签与健康度描述。

## 非目标
- 不调整任务生成、执行与统计等核心流程。
- 不变更与健康度无关的任务/银行数据结构。

## 现状概览
- 标签：存在 `tags` 与 `bank_tags` 表，`/api/v1/tags` 提供列表/创建/删除；银行列表通过标签关联返回 `tags`。
- 健康度：后端通过已完成任务计算 `last_exec_date`，前端在账号管理页计算“健康度”并展示。

## 方案
### 后端
- 移除 `api/v1/tag.go` 与路由注册。
- 删除 `store/model/tag.go` 与 `store/store.go` 中标签相关方法与结构。
- 银行列表改用不含标签的查询；响应结构移除 `tags` 与 `last_exec_date`。

### 前端
- 账号管理页移除标签编辑对话框、标签列、标签入口与过滤逻辑。
- 移除健康度计算函数与列展示。
- 清理 `types`、`api`、`hooks` 中与标签相关的类型与请求。

### 数据库
- 从 `autoMigrate` 中移除 `tags`、`bank_tags`。
- 启动时执行一次性清理：`db.Migrator().DropTable("tags", "bank_tags")`。
- 健康度无独立字段，不涉及表结构删除。

### 文档
- 清理 README/CLAUDE/计划文档中标签与健康度内容。

## 接口与数据流变更
- 删除 `/api/v1/tags` 相关接口。
- `/api/v1/banks` 响应体不再返回 `tags`、`last_exec_date`。

## 风险与回滚
- 标签数据不可恢复；如需回滚需从备份恢复表数据。
- 外部调用方若依赖 `tags` 或 `last_exec_date` 字段需要同步升级。

## 测试与验证
- 账号管理页不再出现标签与健康度列/入口。
- 前端无 `/api/v1/tags` 请求。
- `/api/v1/banks` 响应体不含 `tags`/`last_exec_date`。
- 数据库中 `tags` 与 `bank_tags` 表被移除。
