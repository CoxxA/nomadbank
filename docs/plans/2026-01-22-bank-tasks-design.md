# 银行管理与任务管理页面重构设计

日期：2026-01-22

## 背景与目标
- 背景：当前银行管理页与任务管理页以表格为主，交互与信息层级偏平。
- 目标：在不改变核心流程的前提下，重构为“概览区 + 表格”结构，提升可读性与操作效率。
- 风格：轻量玻璃 + 亮色基底，现代、清爽、信息密度适中。

## 设计原则
- **层级清晰**：主操作区、概览区、筛选区、表格区分层明显。
- **信息聚焦**：高频信息先展示（如下一次转账、待办数）。
- **操作可达**：批量操作在选择后显现，次要操作收纳进更多菜单。
- **性能与简洁**：少量视觉效果（玻璃/阴影）强调分层，不堆叠动画。
- **可读性优先**：对比度达标，行距与留白舒适。

## 视觉系统（轻量玻璃 + 亮色基底）
- 背景：暖白/浅灰基底，可加入轻微渐变或噪点纹理。
- 卡片：轻微模糊 + 1px 细边框 + 低对比阴影，避免厚重玻璃。
- 颜色：主色冷蓝（用于关键按钮/数值），状态色保留绿/橙/灰并提高对比度。
- 字体：正文使用清晰的无衬线；金额/日期使用等宽字体；层级使用更粗的标题字重。
- 交互：hover/active 使用轻微色阶与阴影变化，避免尺寸变形。

## 银行管理页（Accounts）
### 布局结构
1. 标题 + 主操作区（新增、导入、导出、批量动作入口）
2. 概览卡（3–4 张）
3. 筛选栏（搜索、分组、状态）
4. 表格主区域

### 概览卡建议
- 总银行数
- 启用银行数
- 下一次转账（日期 + 目标银行）
- 分组数或待办任务数

### 表格优化要点
- “下次转账”信息紧凑展示：日期 + 时间 + 目标银行 + 金额。
- 状态用胶囊标签（启用/停用）。
- 批量操作条：选择后出现（显示已选数量、批量分组、批量删除）。

### 操作策略
- 主要动作保留主按钮；次要/危险动作收纳在更多菜单。
- 导入/导出按钮保留，增加简短辅助说明（如支持 CSV）。
- 空状态提供“新增银行”引导按钮。

## 任务管理页（Tasks）
### 布局结构
1. 标题 + 主操作区（生成任务、导入、导出）
2. 概览卡（3–4 张）
3. 筛选栏（状态、周期、分组、搜索）
4. 表格主区域

### 概览卡建议
- 待处理数量
- 已完成数量
- 最近完成日期
- 周期分布（可选）

### 表格优化要点
- 核心列：周期、转出/转入、金额、执行时间、备注、状态。
- 行操作（完成/跳过/删除）在 hover 时显现以保持整洁。
- 批量操作条：选择后出现，支持批量删除与按周期清理。

### 生成任务对话框
- 强调策略选择与周期数。
- 保持现有逻辑，仅优化排版层级。

## 数据流与交互
- 数据流沿用现有 Query hooks；统计优先前端聚合，避免新增接口。
- 过滤与分页前端处理（数据规模百级以内，性能可控）。
- 错误处理：toast + 按钮 loading/disabled + 关键操作 confirm。

## 状态与体验
- Loading：轻量骨架或占位行，避免布局跳动。
- Empty：引导操作按钮（生成任务 / 新增银行）。
- Accessibility：对比度达标，焦点状态清晰，按钮可达区域足够。

## 验收要点（手动）
- 筛选组合正确（状态/分组/搜索）。
- 批量操作条显隐与数量统计正确。
- 生成任务流程完整。
- 导入/导出入口清晰、可达。
- 10/100 条数据下列表稳定、无明显卡顿。
