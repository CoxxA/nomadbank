# ADR 0001：单用户 v2 架构

- 状态：已接受
- 日期：2026-07-10

## 背景

NomadBank 面向个人自托管场景。旧版本实现了多用户、角色、成员管理及若干没有后端能力支撑的前端入口，导致认证、数据查询、API 契约和文档维护成本明显高于核心业务本身。

项目尚未正式公开，也没有需要迁移的生产数据，因此 v2 采用破坏式重写。

## 决策

1. 每个部署实例只有一个 owner，首次启动通过初始化页面创建。
2. 使用数据库会话和 HttpOnly Cookie；不再使用 JWT、角色和浏览器可读 Token。
3. v2 使用独立的 `nomadbank-v2.db`，不读取或修改旧版 `nomadbank.db`。
4. SQLite schema 由显式 SQL 创建；后续 schema 变更必须增加版本化迁移。
5. 金额统一使用整数分，数据库主键使用 SQLite 自增整数。
6. 任务生成按批次保存；规划算法不访问数据库，并注入时钟和随机源以便测试。
7. 后端保留 Echo 和纯 Go SQLite，前端保留 React、TanStack Router、TanStack Query 与 Tailwind。
8. OpenAPI 是公开 API 契约和前端类型的唯一来源。
9. v2 只保留真实可用的功能。旧版未实现的导入、延期、跳过、独立 Webhook 与伪批量操作全部删除。
10. 通知功能暂不进入 v2 核心；重新加入时必须同时具备发送、调度、幂等和密钥脱敏。

## 核心范围

- owner 初始化、登录、退出与修改密码
- 银行账户管理
- 保活策略管理
- 任务批次生成、任务列表与完成操作
- 仪表盘摘要
- 单二进制与容器部署

## 结果

- v2 与旧数据库不兼容，旧文件会原样保留。
- 多用户相关 API、表字段、页面和文档全部删除。
- 公开文档只描述当前实现，不展示占位功能。
- 发布前必须通过后端测试、前端测试、类型检查、生产构建和主流程 E2E。
